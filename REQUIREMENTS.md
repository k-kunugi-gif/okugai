## 1. コンセプト

**「FX版・電脳ギルド」— 一人のカリスマではなく、"組織"に所属させる。**

- 総帥の下に専門分野の異なるメンバーが集い、「億り人・稼ぐ人を増やす」というミッションを掲げた結社型コミュニティ
- 新宿租界から移植するもの: 組織の世界観 / 総帥+ロスター構造 / キャラごとの専門とファン化 / ポータル型HP / 事業の多角化を組織の「部門」として見せる演出
- 移植しないもの: **予想（シグナル）の販売** — 金商法上の投資助言に該当するため、各メンバーの提供物は「検証レポート・教育・ツール」に置換する（詳細は §7）

### ミッションコピー候補（未決）
- 「億り人を、量産する。」
- 「勝てる個人を、組織で創る。」
- 「相場で食う技術を、継承する。」

## 2. ブランド・ネーミング（✅確定 2026-07-11）

**サロン正式名称: 「億街サロン」**（2026-07-11変更。旧表記「億街 -秘密基地-」）

```
億街（オクガイ）      ← 親ブランド・世界観「億り人が住む電脳の街」・HP＝街の公式ポータル
 ├─ 億街サロン       ← サロン正式名称（対外・商品表記。LP/特商法/©表記）
 │    └─ 「秘密基地」 ← Discord拠点の世界観上の通称（縁起由来・「基地に集合」）
 └─ 億荘（オクソウ）  ← サロン内の上位レイヤー（選抜入居制・集中育成）。§2.5参照
```

- 世界観ルール: **運営者（総帥+AIキャラ+専門家）＝秘密基地の「住人」**。会員は「移住者/住民」、入会＝「移住」、キャラ追加＝「新しい住人が越してくる」
- Discordチャンネル＝街・基地の施設（司令室=お知らせ、検証室、工房=ツール、酒場=雑談 等）
- 将来部門の拡張名も「街の施設」として命名（工房=ツール販売 / 億荘=上位育成レイヤー / 展望塔=メディア連携 / 億横丁・地下億階等は拡張予備）
- ドメイン: **okugai.jp（推奨・要取得）**。空き確認済み: okugai.jp / oku-gai.com / okugai-fx.com / okugaikichi.jp / okugaikichi.com / okusou.jp（okugai.com・okusou.comのみ第三者取得済み）
- ボツ候補ログ: ALMA GUILD / EDGE SYNDICATE / 億番地 / 億横丁 / 地下億階（施設名として再利用可）

### 2.5 億荘（オクソウ）— 上位商品の設計（✅方向性確定 2026-07-11: B案×C案ミックス）

**「億街の外れに建つ、億り人が量産されると噂のボロ(実は電脳の)アパート」** — トキワ荘の投資版。

| 項目 | 設計（数値は仮・要決定） |
|---|---|
| 位置づけ | 億街サロンの上位レイヤー。旧「訓練場(スクール)」を置換 |
| 入居方式 | **選抜・入居制**。応募条件例: 正住民歴1ヶ月以上+検証ログ提出。面談(審査)あり |
| 定員 | **全6部屋**(101〜103・201〜203号室)。「203号室、空きました」と募集する |
| 料金 | **家賃 月29,800円(仮)** のサブスク(C案要素)。買い切り講座ではなく毎月の家賃 |
| 部屋 | 各入居者にDiscord個人スレッド=「自分の部屋」。検証ログを公開で積み、住人(運営)が技術面をフィードバック |
| カリキュラム | FXを勉強したい/稼ぎたい/極めたい人向けの**集中指導**——トレード技術・検証・資金管理+実装(EA/ツール自作、Python/MT4/MT5)。※技術指導に限定し、個別の売買指示は行わない |
| 卒業 | 卒業要件(例: 自作検証ツール完成+検証レポートN本)を満たすと**「億荘◯号室 卒業生」**の称号。卒業がゴールなので長期滞留させない設計 |
| 物語効果 | 卒業生が増えるほど「億荘出身」がブランド化(トキワ荘構造)。部屋の空き=希少性、入居募集が告知イベントになる |
| ⚠️ 法規制 | 個人スレッドでのフィードバックは**技術・検証手順・資金管理の教育に限定**。「このエントリーは正解/不正解」等、個別トレードの善し悪しの指導は投資助言に接近するため行わない(事後の検証方法を教えるのはOK)。成果保証表現禁止は基地と同様 |

ファネル完成形: **億街(無料・HP/SNS) → 億街サロンに移住(月980/4,980円) → 億荘に入居(選抜・月29,800円) → 卒業(称号・実績) → 億ヒルズに認定招待(§2.6)**

### 2.6 億ヒルズ — 招待制の最上階レイヤー(✅方向性確定 2026-07-11)

**「街の中心にそびえるタワー。金では入れない。認定でしか入れない」** — 億荘(ボロアパート=修行)との対比構造。

| 項目 | 設計 |
|---|---|
| 位置づけ | 街の最上位レイヤー。**販売しない(非売品)**。招待制・認定制 |
| 入居条件 | 億荘の卒業生、または運営が実績を確認できた住民を招待。**実績は自己申告ではなく運営の確認(口座履歴等)を経る**(優良誤認回避) |
| 中身 | 上級者限定ラウンジ(クローズドch)、顧問(実在専門家)との接点、後進(億荘入居者)のメンター役 |
| 収益効果 | 直接収益ではなく、①金で買えない最上位の存在が下位商品の信頼を担保 ②住民=生きた成功事例として集客エンジン ③将来は工房ツール共同開発・講師役のレベニューシェアに発展可 |
| 世界観の階段 | 移住(億街サロン) → 入居(億荘) → **認定(億ヒルズ)** — 全ステップが「住まいの格上げ」 |

## 3. ロスター設計（ドラフト）

FX特化のため、新宿租界の「競馬/競艇/競輪」に相当する専門分化はFX内の「時間軸・アプローチ」で行う。

> ⚠️ 2026-07-11改訂: **AIはアルマ一体のみ。他の住人(ブリッツ/ノクス/ナギ)は人間**(仮名で活動する人間トレーダー。実在者のリクルートまたは運営本人)。下表の「正体」列は旧版のため、アルマ以外の「AIキャラ」は「人間」に読み替えること。

| 役職 | 正体 | 専門 | キャラ案（仮名・変更前提） | 主な提供物 |
|---|---|---|---|---|
| 総帥 | ユーザー本人（匿名キャラ） | 組織運営・AI×投資の思想 | コードネーム未決 | ビジョン発信、統括 |
| メンバー1 | AIキャラ | スキャル/短期テクニカル | ブリッツ | 短期手法の検証レポート・教育 |
| メンバー2 | AIキャラ | スイング/ファンダメンタルズ | ノクス | 中長期分析の考え方・事後解説 |
| メンバー3 | AIキャラ | クオンツ/バックテスト | アルマ（既存キャラ継承） | 検証ツール・インジケータ開発 |
| メンバー4 | AIキャラ | 資金管理・統計・メンタル | 未定 | 資金管理教材・破産確率等の教育 |
| 顧問枠 | **実在の専門家**（随時参画） | 現役トレーダー/税理士等 | 実名 or 公認ペンネーム | 監修・対談・スクール講師 |

### ロスターの信頼設計ルール（必須）
1. AIキャラは**AIであることをHP上で公表**する（「電脳メンバー」等、世界観に織り込む）。架空の経歴・架空のトレード実績は一切持たせない
2. 実在専門家の実績・肩書は**本人確認できる事実のみ**記載（優良誤認の回避）
3. コンテンツの最終監修責任者（=運営者）をHPに明記

## 4. サイト構成

### 4.1 HP（組織ポータル）`hp/`

```
トップ
├─ HERO: 組織ロゴ+ミッション+世界観ビジュアル
├─ ABOUT: 組織の物語（なぜ結成したか/何を目指すか）
├─ MEMBERS: ロスター一覧 → キャラ個別ページ（or モーダル）
│    └─ 各キャラ: ビジュアル/専門/提供コンテンツ/SNSリンク
├─ DIVISIONS(事業部門): サロン / ツール開発 / スクール / メディア(fx-media)
│    └─ 各部門カード → サロンはLPへ導線
├─ ARCHIVE: 検証レポート/活動実績のサンプル公開（信頼の証拠）
├─ JOIN: 入会LPへのCTA
└─ FOOTER: 特商法/免責/プライバシーポリシー/運営者情報
```

### 4.2 LP（サロン入会用）`lp/`

既存ドラフト（index.html）の骨格を流用しつつ組織型に改修:

```
HERO(組織名+ミッション+CTA)
→ 問題提起(独学・聖杯探し・シグナル依存の限界)
→ 解決策: 「組織に所属する」という選択肢
→ ロスター紹介(HPより簡略なカード列)
→ 提供コンテンツ(検証レポート/ツール/教育/Discord)
→ 料金プラン(ライト980円/スタンダード4,980円 ※仮)
→ FAQ(シグナル配信なし明言/AIキャラ公表/解約)
→ リスク警告・免責
→ 最終CTA(事前登録)
→ 特商法リンク
```

### 4.3 デザイン要件（✅実装済み 2026-07-11 デザインシステムv2）

- **タイポグラフィ**: 見出し・縁起=Shippori Mincho(明朝・文芸/高級感)、UI=Zen Kaku Gothic New(Google Fonts)

## 5. 機能要件

| フェーズ | 機能 |
|---|---|
| P0（完了） | 静的HP+LP、特商法ページ。事前登録導線（LINE公式 or フォーム）は受け口未決 |

**ログイン機能の設計判断（2026-07-11確定）**: 独自ログイン・パスワード管理はサイトに持たせない。会員エリア=Discordであり認証はDiscordが担う。サイト側はStripe決済→Discord OAuth連携のみ（カード情報・パスワードを一切保持しない=セキュリティ/個人情報リスク最小）。Web上の住民限定アーカイブが必要になった段階でSupabase Auth(Discordログイン)を追加する二段構え。

### 5.5 顧客データ設計(CRM方針・2026-07-12)

専用CRMは導入しない。**既製の3台帳+自前ハブDB1個**で運用する。

| 台帳 | 役割 |
|---|---|
| LINE公式 | リード台帳(回覧板登録者)。タグ機能でセグメント管理 |
| Stripe | 課金台帳(顧客・サブスク・LTV・解約)。実質メインCRM |
| Discord | 会員名簿(ロール=仮住民/正住民/億荘/卒業生/ヒルズ) |
| 自前DB | 3台帳の紐付けハブ: subscribers(email/line_id/utm) / members(stripe_customer_id⇄discord_id/plan/status) / events(行動ログ) |

- 現在はbackend/server.jsのJSONストア(開発用)。**デプロイ時にSupabase(推奨: 無料Postgres+管理画面=CRM画面不要)またはSQLiteに確定**
- カード情報は永久に自前保持しない(Stripe徹底)。DB委託先確定時にプライバシーポリシー第3条へ追記
- 会員100人超で検討: LINEタグのセグメント配信、専用管理ダッシュボード(~/Projects/crmの技術構成を流用可・スキーマは要差し替え)
