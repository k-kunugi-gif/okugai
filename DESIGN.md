## 1. 全体動線マップ(入り口 → 収益 → 上位階層)

```
【入り口(流入チャネル)】
 X(キャラ運用) ─┬─→ 固定ポスト(縁起) ─→ HP ─→ LP
 YouTubeショート ─┤                          ↑
 展望塔(自社SEO) ─┤─→ コラム/比較 ─→ アフィ収益 + LP送客
 fx-media(海外FX・別ドメイン) ─→ 記事内CTA ─→ LP
 GogoJungle ─────┤─→ ツール購入者向け案内 ──→ LP
 指名検索/紹介 ──┴─→ HP(受け皿) ─→ LP

【転換(フェーズで切替)】
 P0 事前登録期:  LP ─→ LINE公式 登録(無料) ……「事前登録者リスト」
 P1 募集開始後:  LINE配信 ─→ LP ─→ Stripe Checkout ─→ /welcome
                                                        └→ Discord OAuth ─→ 秘密基地 入室+ロール付与

【入室後(サロン内動線 = 収益の階段)】
 新入りの路地(オンボーディング) → 鍛錬場/検証室(定着) → 工房(ツール単品購入)
   → 億荘 入居応募(選抜) → 卒業 → 億ヒルズ 認定(招待)
 退去: Stripe Billing Portal(自己完結) → Webhookでロール剥奪
```

### 動線の原則
1. **HPは「信じさせる」、LPは「動かす」**。迷ったらCTAはLPへ、物語はHPへ
2. **すべての外部発信の着地はまずLP**(HPは指名検索・プロフィールリンクの受け皿)
3. **転換は常に1つ**: P0では「LINE登録」だけ、P1では「決済」だけ。CTAを競合させない

## 2. ページ別仕様

### 2.1 HP(hp/index.html) — 億街ポータル
- **役割**: 世界観への没入と信頼構築。指名検索(「億街サロン」「億街 FX」)の受け皿。SNSプロフィールのリンク先
- **主要KPI**: LP送客率(HP訪問→LP遷移)
- **セクション構成と目的**:

| セクション | 目的 | 期待アクション |
|---|---|---|
| HERO(キービジュアル) | 3秒で世界観を提示 | スクロール or 「移住する」→LP |
| STORY(縁起) | 物語で価値観に共感させる | 読了 |
| RULES(掟) | 法令順守を誠実さとして提示、疑念の除去 | — |
| RESIDENTS(住人名鑑) | キャラへの興味・AI公表の透明性 | 推し住人の発見 |
| FACILITIES(施設) | 街の全体像と将来の階段(億荘/ヒルズ)の可視化 | サロンカード→LP |
| COMPLIANCE | リスク明示(法対応) | — |
| JOIN | 最終送客 | →LP |

### 2.2 LP(lp/index.html) — 移住案内(セールスページ)
- **役割**: 唯一の転換装置。P0=LINE登録、P1=Stripe決済
- **主要KPI**: P0=訪問→LINE登録CVR(目標10%)、P1=訪問→決済CVR(目標2〜4%)
- **説得の階段(セクション順の意図)**:

| 順 | セクション | 説得上の役割 |
|---|---|---|
| 1 | HERO+住民カード | 世界観+何が得られるかの即時提示、第1CTA |
| 2 | 縁起引用帯 | 価値観の宣言(「予想は売らない」) |
| 3 | PROBLEM | 痛みの言語化(聖杯/シグナル/孤独)=自分ごと化 |
| 4 | DIFFERENCE(比較表) | 「稼がせる系」との決別。掟を競合優位に変換 |
| 5 | SOLUTION+レポートモック | 提供物の具体化。モックで中身への好奇心(数値はロック) |
| 6 | RESIDENTS | 誰から学ぶか(AI公表の透明性) |
| 7 | BASE MAP | 入った後の生活のイメージ形成 |
| 8 | WEEKLY RHYTHM(基地の一週間) | 入会後の時間の流れを具体化。「張り付き不要」の証明 |
| 9 | FIT CHECK(適性確認) | 来てほしい人/向いていない人の明示。断る勇気=信頼、不適合客の事前排除 |
| 10 | PRICING+初期住民特典 | 価格提示と今すぐ動く理由(希少性) |
| 11 | FLOW(3ステップ) | 手続き不安の除去 |
| 12 | FAQ | 反論処理(シグナル無し/AI/初心者/多忙/保証しない/解約) |
| 13 | COMPLIANCE | リスク明示 |
| 14 | FINAL CTA(夜景+安心チップ3種) | 感情のピークで最終CTA。無料/退去自由/シグナル無しの最終不安除去 |
- **CTA配置**: ヘッダー/HERO/料金カード×2/FINAL/モバイル追従 = 常時1タップ圏内

### 2.5 展望塔(media/) — メディア・アフィリエイト
- **役割**: ①SEO流入の入り口(コラム/比較) ②アフィ収益(FX口座開設、1件2〜4万円) ③サロン送客(全ページにLP導線)
- **構成**: index.html(塔トップ=コラム一覧+比較導線) / fx-hikaku.html(国内FX比較・主要6社) / kaigai-fx.html(海外FX比較・主要5社) / column-XXX.html(住人執筆コラム連載)
- **編集方針**: コラムは住人キャラ執筆+「AIキャラ執筆・運営検証監修」を明記。個別売買推奨なし。**海外FXも展望塔内で扱う(2026-07-11方針変更・fx-media隔離案を撤回)**。ただし海外ページは冒頭に赤枠リスク警告(金融庁無登録・出金リスク・総合課税最大55%)を必置し「利点も危険も全部書く」編集方針を掟として明文化。国内⇄海外の相互導線あり(初心者は国内へ誘導)
- **アフィ規制対応**: 全アフィページ冒頭に「本ページはプロモーションを含みます」バナー(ステマ規制) / リンクにrel="sponsored" / スペック数値は公開前に公式で確認(現在は傾向表記+TODOバナー)
- **主要KPI**: 記事→LP遷移率 / アフィCTAクリック率 / 検索流入数
- **コラム→サロンの動線設計**: 記事末尾に「この続きは、街の中で」CTA(教育コンテンツの深い部分=住民限定、が基本構造)

### 2.5.5 街の告知板(HP)の更新方法
- **data/news.json から自動生成**(2026-07-12実装)。HPの読み込み時にJSが描画、フェッチ失敗時はHTML内の静的リスト(旧4件)にフォールバック
- 更新手順: news.json の**配列先頭**に `{date, datetime, tag, type("open"=緑/"soon"=金/"info"=シアン), html(相対リンクは../起点)}` を1件追加するだけ。HTMLは触らない
- 運用ルール: Claudeが街に大きな変更(コラム公開/募集開始/施設オープン等)を加えた際は、作業とセットでnews.jsonにも追記する

## 4. 流入チャネル設計(UTM規約)

- URL規約: `?utm_source={x|youtube|fxmedia|gogojungle|line}&utm_medium={social|seo|referral|crm}&utm_campaign={launch|founding|weekly}`
- 例: Xプロフィール→ `lp/?utm_source=x&utm_medium=social&utm_campaign=profile`
- チャネル別の着地:
  - X/YouTube(キャラ運用) → 通常ポスト=LP直 / 固定ポスト=縁起(HP)→LP
  - fx-media記事 → 記事文脈に合わせLP(PR表記必須)
  - GogoJungle商品ページ → 販売者プロフィール経由でHP(規約上直リンク制限があれば)

## 5. 計測設計(GA4 — ホスティング時に導入)

| イベント | 発火箇所 | 見る指標 |
|---|---|---|
| `cta_register_click` | 全登録CTA(位置をparamで区別: hero/pricing/final/sticky) | どの位置のCTAが効くか |
| `cta_checkout_click` | P1の決済ボタン | プラン別クリック率 |
| `scroll_depth` | 25/50/75/100% | LPの離脱ポイント(比較表前後を重点監視) |
| `faq_open` | details開閉 | 反論の頻度(=コピー改善のネタ) |
| `outbound_line` / `outbound_discord` | 外部遷移 | 転換の実数 |
- ファネル定義: 訪問 → scroll50 → cta_register_click → LINE登録(LINE側計測) → (P1)決済 → Discord入室
- 目標値(初期仮説): LP訪問→LINE登録 10% / LINE登録→初期住民決済 15% / 決済→Discord入室 98%
